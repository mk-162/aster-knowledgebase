# Organiser quick-start checklist

Your first event, start to finish. Each step links to the detailed guide. Work
top to bottom and you'll have a fully set-up, race-ready event.

## A. Before you build the event

- [ ] **Create your organiser account** — name, handle, logo, description.
      [Become an organiser](become-an-organiser.md)
- [ ] **Add anyone who'll help** as an admin of the organiser account, so race-day
      duties can be shared.
- [ ] **Have your route ready** as a GPX file.
- [ ] **Know your timing**: the exact start date, **time** and the event's local
      **time zone**.

## B. Build the event

- [ ] **Choose the host** (this organiser account) — it can't be changed later.
- [ ] **Pick the event type** — race, time trial, or group ride / adventure.
      [Event types explained](event-types.md)
- [ ] **Upload the route (GPX)** and check the distance and elevation look right.
      [Routes, start and finish lines](routes-start-and-finish.md)
- [ ] **Confirm the start and finish lines** the route suggested — adjust the
      positions and detection radius if needed.
- [ ] **Set the start time and time zone.** Double-check the zone.
- [ ] **Set a cut-off** if you need one, and choose whether timed-out riders are
      marked DNF or finished.
- [ ] **For circuits:** set the lap count and lap-detection zone.
- [ ] **Choose privacy** — public, unlisted or private.
      [Public, unlisted and private events](../for-spectators/public-unlisted-private-events.md)

## C. Registration

- [ ] **Set how riders join** — open or by approval.
      [Registration and categories](registration-and-categories.md)
- [ ] **Add your categories** (with wave start times if you use them).
- [ ] **Set capacity** and the registration open/close window if you need them.
- [ ] **Build the registration form** — questions, emergency contact, waiver.
- [ ] **Choose bib assignment** — automatic or self-entry.
- [ ] **Publish**, then open the event page yourself to check it reads well.

## D. In the week before

- [ ] **Share the event link** (and access code, if private).
- [ ] **Post a welcome update** to the event feed.
- [ ] Remind riders to **install the app**, **register**, and open the event
      **once with signal** so it's cached for an offline start.
      [Register for an event](../for-athletes/register-for-an-event.md)
- [ ] Confirm your **on-the-ground emergency plan** — Watch the Dot's SOS is an
      alert and location share, not a substitute for calling for help.

## E. Race day

- [ ] Open your **organiser dashboard**. [Running the event on race day](race-day-management.md)
- [ ] Check the **participant list** — riders showing as registered/started.
- [ ] **Start the race** at the gun (mass-start). If you forget, it auto-starts
      shortly after the scheduled time, but press it yourself for precision.
- [ ] **Watch the field** on the live map and leaderboard.
- [ ] **Keep an eye out for SOS alerts** and respond. [Handling SOS alerts](handling-sos-alerts.md)
- [ ] Use **force finish / mark DNF / reset** for any rider whose tracking needs a
      hand.

## F. After the finish

- [ ] **Publish results.** [Results](results.md)
- [ ] If you use chip timing, **import** the timed results.
- [ ] **Export results (CSV)** for your records or your federation.
- [ ] **Post a recap** to the event feed and thank everyone.

> **Good to know:** once a rider finishes (or is cut off), their result freezes —
> late GPS data never changes a finisher's time or position, so your results stay
> clean.
