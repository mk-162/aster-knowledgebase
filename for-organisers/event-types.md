# Event types explained

The event type you choose changes how riders start, how they're timed, and how
they finish.

## Mass-start race

Everyone starts together on a **gun**. You (or the scheduled start time) fire the
gun; every rider's clock starts from that same moment. Riders are ranked by
progress along the course, and the first across the finish line wins. Riders are
finished automatically when they cross the finish line.

Best for: criteriums, road races, mass-start gravel events.

## Time trial

Riders start **individually** and each is timed on their **own clock** from when
they cross the start. Standings are by fastest elapsed time. Time trials can show
**split times** at checkpoints so you (and spectators) can see who's fastest on
each section.

Best for: individual or team time trials, hill climbs.

## Group ride / adventure (point-to-point)

Started individually, usually social or self-supported rather than competitive.
These suit long-distance "grand départ" style adventures. You can place
**checkpoints** along the route as waypoints.

Best for: group rides, bikepacking grand départs, social outings.

## How finishing differs

| | Finish detection | Timed from |
|---|---|---|
| Mass-start race | Crossing the finish line | The gun |
| Time trial | Crossing the finish line | Each rider's own start |
| Group ride / adventure | Crossing the finish (or final checkpoint) | Each rider's own start |

In every type, a rider can also **finish or abandon manually**, and your
**cut-off** (if set) closes out anyone still riding past the deadline. See
[Finishing a ride](../for-athletes/finishing-a-ride.md).

## Laps

For circuit events you can set a **lap count** and a lap-detection zone, so the
app counts laps as riders pass the line.
