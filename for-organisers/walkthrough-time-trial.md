# Walkthrough: a time trial

In a time trial (TT), riders start **one at a time** and each races their **own
clock**. The fastest elapsed time wins. This walkthrough covers setting one up
and getting the most from split times.

## 1. Create the event as a "time trial"

Choose the **time trial** type when you [create the event](create-an-event.md).
Each rider's clock starts when they cross the start, not on a shared gun.

## 2. Route, start and finish

[Upload your GPX](routes-start-and-finish.md) and set the **start** and **finish**
lines. The start line matters more here than in a mass start, because each
rider's time begins the moment they cross it — place it where riders are up to
speed and the line is unambiguous.

## 3. Splits with checkpoints

To get **split times** (and let spectators compare riders section by section),
the course can have checkpoints along it. Splits make a TT far richer to follow —
viewers can see who's fastest on the climb versus the flat. See
[Comparing riders](../for-spectators/comparing-riders.md).

## 4. Categories and start order

Add your [categories](registration-and-categories.md). TTs are usually run as a
start list — riders set off at intervals — so think about your start order and,
if you use waves, set **wave start times** per category.

## 5. On the day

- Riders **start individually** and are timed from their own start crossing.
- They're **finished automatically** at the finish line.
- The leaderboard ranks by **fastest elapsed time**, and gaps compare each
  rider's own time — so a rider who started later can still top the board.
- Watch for **SOS alerts** as usual. [Handling SOS alerts](handling-sos-alerts.md)

## 6. After: results

[Publish results](results.md) — ranked by time within each category. Import chip
times if you have them, and export a CSV.

## Things to get right

- **Start-line placement is the timing point.** Each rider's clock begins at the
  start crossing, so make the line clean and consistent for everyone.
- **Use checkpoints for splits.** They turn a TT from a single number into a
  segment-by-segment story for spectators.
- **Elapsed time, not finish order, decides it.** The first rider to physically
  finish isn't necessarily the winner — the fastest time is.
